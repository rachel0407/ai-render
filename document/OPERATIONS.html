一、這個系統在做什麼

讓使用者匿名打開頁面 → 選一個產品 → 上傳自己的設計圖（LOGO / 照片）→ 系統自動去背 → 拖拉調整位置與大小 → AI 把設計「印」到產品上 → 預覽 OK 直接下載。

整個流程不需要註冊、不需要登入；所有渲染紀錄會保留在後台供查驗。

二、流程（自動）

步驟系統行為使用者看到
1. 使用者開首頁 (/) 拉產品資料夾清單 左側產品列
2. 選產品、上傳設計圖 後端自動去背、回傳透明 PNG 縮圖 + 拖拉框
3. 拖拉/縮放定位、點「開始渲染」 前端把底圖+設計合成 → 送 Gemini → 自動寫一筆歷史 約 15–30 秒後出結果
4. 點「下載結果」 直接下載 PNG —
✅ 沒有結帳、購物車、訂單這些概念 每一次渲染就是一筆獨立紀錄。

三、後台 admin 使用

登入

  • 網址：https://<host>/admin
  • 密碼：找工程師設（用 bcrypt 寫進 .env，沒有預設密碼）
  • 8 小時自動登出

後台有兩個 tab

Tab 1：歷史紀錄

  • 依時間倒序列出每一筆渲染
  • 每筆顯示：日期時間、使用者上傳的原圖、AI 合成結果、用到哪個產品資料夾
  • 點圖片可放大、可下載
  • 上方有「渲染總數 / 本頁顯示 / 當前頁」三個小計
  • 預設一頁 24 筆，下方分頁按鈕翻頁

Tab 2：資料夾管理

  • 新增資料夾：右上「+ 新增資料夾」→ 輸入名稱（限英數字 / 底線 / 連字號，1–64 字元）
  • 上傳底圖：每張卡片右側「上傳底圖」，可一次選多張（單檔上限 50 MB）
  • 刪除底圖：圖片下方的「刪除」
  • 刪除整個資料夾：卡片右上的「刪除資料夾」（會清掉內部所有底圖，不可復原）

底圖檔名小規則

檔名包含用途
(正) 或 （正）設為這個資料夾的預覽圖；雙面模式的正面也用這張
(背) 或 （背）雙面模式的背面
都沒有取第一張當預覽 / 正面；雙面時背面 fallback 用預覽圖

四、資料保留規則

不複雜 — 全部永久保留。

source_image/產品底圖（後台上傳的）永久
upload_image/使用者原始上傳的設計圖永久
result_image/AI 合成結果永久
history.jsonl渲染紀錄索引（時間、檔名對應）永久
⭐ 全部永久保留 空間不夠時請工程師清理。沒有自動 TTL，也沒有訂單概念。

五、客服常見問題對應

Q1. 「我渲染出來的圖在哪裡？」
回應渲染完成後右下角會有「下載結果」按鈕，點下去就會存到您的下載資料夾。系統不會自動保存到您的裝置上。
Q2. 「我之前渲染過的圖，可以再下載一次嗎？」
判斷使用者端是匿名的，本身沒有「我的紀錄」這個功能。
做法請使用者大約告訴您時間 + 用的哪個產品資料夾；您進後台「歷史紀錄」用分頁找到那筆，下載合成結果回傳給他。
想要做使用者自助查詢的話，要排開發。
Q3. 「我能不能傳很大張的圖？」
回應可以，但建議單張小於 10 MB（瀏覽器壓縮會比較快）。後端去背的處理時間跟圖的尺寸成正比，超過 1600px 邊長會先縮圖。
Q4. 「為什麼 AI 合成結果看起來不太像？」
判斷
  • 設計圖是不是被自動裁過邊？（系統會自動裁透明邊 + 近白邊）
  • 拖拉框有沒有放在合理位置？（出可印刷範圍框會被限制）
  • 底圖品質夠不夠？（解析度太低或角度怪，Gemini 會抓不到光影）
回應建議使用者重新調整位置/大小再渲染。或請工程師檢查該資料夾的底圖品質。
Q5. 「我看到後台歷史紀錄裡有違法 / 侵權的內容」
做法
  1. 截圖保留證據
  2. 直接從伺服器刪掉對應的 upload_image/{檔名} 與 result_image/{檔名}（請工程師處理或自己 ssh 進去）
  3. 編輯 storage/history.jsonl 把那一行刪掉
  4. 必要時通報主管 / 法務
Q6. 「我上傳了底圖但使用者看不到」
判斷
  • 後台「資料夾管理」裡，那個資料夾的「N 張底圖」數字有沒有更新？沒更新 → 上傳失敗
  • 數字有更新但使用者端沒看到 → 請使用者 Cmd+Shift+R 重整（瀏覽器快取）
回應請對方按 Ctrl+Shift+R（Win）或 Cmd+Shift+R（Mac）強制重整。

六、需要找工程師的情況

狀況為什麼
後台 admin 進不去（密碼正確）後端 / Token 問題
渲染一直失敗（多個使用者反映）Gemini quota 滿 / API key 失效
想清理過舊的 upload_image / result_image沒有自動 TTL，得手動排程
看到使用者設計圖內容違法 / 侵權要手動刪 + 留紀錄 + 通報
想加 user 端的「歷史紀錄」/ 自助下載功能還沒做，要排開發
想串接其他系統（例如電商加購物車）API 已備好，要寫整合層
storage/ 磁碟快滿要決定保留策略（永久 vs TTL）

七、安全注意事項

⚠️ 首次部署一定要做
  1. 設一組強密碼，bcrypt 寫進 .env 的 ADMIN_PASSWORD_HASH
  2. 隨機產 32+ 字元的 ADMIN_SESSION_SECRET（漏了等於有人可以偽造登入 token）
  3. 細節看 SECURITY.md

管理員自己要注意

  • 後台密碼不要寫在便條紙 / Slack / Email 明碼傳遞
  • 後台 token 有效期 8 小時，下班前可順手登出（不登出也沒事，會自動過期）
  • 開後台時注意身邊有沒有人偷看（會看到所有使用者上傳的設計圖）
⚠️ 資安已知缺口 /upload/*、/result/*、/source/* 任何人知道檔名就能下載（沒擋 unauthenticated GET）。
影響範圍：被外洩的單張圖無法影響系統其他資料，但客戶設計圖的隱私沒有完全保護。修法在 SECURITY.md 待辦清單。
